# Lemmings Clone — Техническое задание

> **Проект:** Прототип игры Lemmings (клон классики 1991 года)
> **Целевые платформы:** iOS + Android
> **Движок:** Godot Engine 4.x (GDScript)
> **Язык:** Русский (интерфейс), English (код)

---

## 1. Обзор оригинальной игры

### 1.1 Корневой геймплей

Lemmings — пазл-стратегия, где игрок управляет потоком антропоморфных леммингов, выходящих из люков-порталов. Цель: довести заданный процент популяции до выхода, назначая ограниченный набор навыков отдельным особям.

| Параметр | Оригинал (1991) |
|----------|-----------------|
| Уровней | 120 (4 категории × 30) |
| Категории сложности | Fun → Tricky → Taxing → Mayhem |
| Навыков | 8 типов |
| Режимы | Одиночный + два игрока (split-screen) |
| Таймер | На каждый уровень |
| Разрешение | 320×200 (Amiga) → адаптивное под мобильные |
| Управление | Мышь → Touch |

### 1.2 Восемь навыков (Skills)

| # | Навык | Оригинал | Действие |
|---|-------|----------|----------|
| 1 | **Climber** | Climber | Лемминг карабкается по вертикальным стенам. Падает при встрече с выступающим карнизом. |
| 2 | **Floater** | Floater | Раскрывает парашют при падении. Безопасная высота падения — любая. |
| 3 | **Bomber** | Bomber | Взрывается через 5 секунд. Уничтожает себя и ландшафт в радиусе. НЕ наносит урон другим леммингам. |
| 4 | **Blocker** | Blocker | Стоит на месте, блокирует проход. Встречные лемминги разворачиваются. |
| 5 | **Builder** | Builder | Строит лестницу из 12 ступенек вверх. Останавливается при ударе головой, нехватке кирпичей или встрече с твёрдым объектом. |
| 6 | **Basher** | Basher | Роет горизонтальный туннель. Не может пробить сталь. |
| 7 | **Miner** | Miner | Роет диагональный туннель вниз. Не может пробить сталь. |
| 8 | **Digger** | Digger | Роет вертикально вниз. Не может пробить сталь. |

### 1.3 Механики лемминга

- Лемминг по умолчанию идёт в одном направлении
- При встрече с препятствием разворачивается
- При встрече с Blocker'ом разворачивается
- Падение с высоты > определённого порога = смерть
- Падение за нижний край экрана = смерть
- Вода, лава, огонь, ловушки = смерть
- Присвоенный навык действует до завершения анимации или прерывания

### 1.4 Управление уровнем

- **Release Rate:** 1–99 (скорость выхода леммингов из люка)
- **Nuke:** превращает всех оставшихся леммингов в Bombers
- **Pause:** пауза (можно осматривать уровень, нельзя назначать навыки)
- **Fast-forward:** ускорение времени (современное дополнение)
- **Restart:** перезапуск уровня

### 1.5 Условия победы/поражения

- **Победа:** спасено ≥ требуемого процента леммингов (до истечения таймера)
- **Поражение:** таймер истёк ИЛИ недостаточно леммингов для выполнения квоты
- При достижении квоты до истечения таймера — уровень завершается досрочно

## 2. Анализ опыта игроков

### 2.1 Ветераны 90-х (целевая аудитория 1)

**Что ценят:**
- Аутентичный pixel-art стиль
- Оригинальные звуки (Let's go!, Oh no!, Yippee!)
- Тот же баланс сложности (Fun → Mayhem)
- Мышечную память на тайминги навыков
- 120 оригинальных уровней
- Отсутствие pay-to-win, рекламы, energy-механик

**Болевые точки оригинала:**
- Невозможность отменить назначенный навык
- Отсутствие реплея/перемотки
- Привязка к конкретному разрешению
- Нет undo для навыков

### 2.2 Новая аудитория

**Ожидания:**
- Интуитивное touch-управление
- Визуальная привлекательность (современный pixel-art)
- Постепенное обучение (tutorial)
- Система подсказок
- Достижения (achievements)
- Сохранение прогресса
- landscape + portrait режимы

**Чего не ждут:**
- Сложных меню
- Текстовых инструкций (всё через геймплей)

### 2.3 Синтез: дизайн-решения

| Аспект | Решение |
|--------|---------|
| Визуальный стиль | Современный pixel-art (16×16 спрайты, масштабирование под экран) |
| Управление | Tap для выбора навыка, tap по леммингу для назначения. Кнопки навыков — нижняя панель. |
| Сложность | 120 оригинальных уровней + режим песочницы |
| Обучение | Первые 5 уровней Fun = интерактивный туториал |
| Монетизация | **Нет.** Разовая покупка (premium). Без рекламы, без IAP. |
| Сохранения | Автосохранение прогресса. 3 слота. |
| Достижения | Game Center (iOS) / Google Play Games (Android) — опционально |

## 3. Архитектура

### 3.1 Выбор движка: Godot Engine 4.x

**Обоснование:**
- ✅ Бесплатный, open-source (MIT)
- ✅ Нативная кроссплатформенность: iOS + Android из одной кодовой базы
- ✅ Отличный 2D-рендеринг (TileMap, Parallax, AnimationPlayer)
- ✅ GDScript — низкий порог входа
- ✅ Активное сообщество, документация
- ✅ Существующие примеры Lemmings-клонов на Godot
- ✅ Нет лицензионных отчислений (в отличие от Unity)
- ✅ Лёгкий экспорт под мобильные платформы

### 3.4 Формат данных уровней

Уровни хранятся в текстовом формате (JSON или INI-подобный):

```ini
[level]
name = "Just Dig!"
difficulty = "fun"
number = 1

[goals]
save_percentage = 50      # % для спасения
time_limit = 300           # секунд
lemming_count = 80         # всего леммингов
release_rate = 50          # начальная скорость выхода

[skills]
climber = 0
floater = 0
bomber = 0
blocker = 0
builder = 0
basher = 0
miner = 0
digger = 10

[layout]
# TileMap данные или ссылка на .tscn
scene = "res://levels/fun/level_01.tscn"
```

### 3.5 Конечный автомат лемминга

```
                  ┌──────────┐
                  │  SPAWN   │
                  └────┬─────┘
                       ▼
               ┌──────────────┐
         ┌─────│   WALKING    │◄────────────┐
         │     └──┬──┬───┬───┬┘             │
         │        │  │   │   │              │
    ┌────▼───┐ ┌──▼──▼─┐┌──▼───┐┌──────────▼──┐
    │CLIMBING│ │FALLING││DYING ││SKILL_ACTIVE │
    └────┬───┘ └──┬────┘└──────┘│(build/bash/ │
         │        │             │ mine/dig)   │
         │  ┌─────▼──────┐      └──────┬──────┘
         │  │  FLOATING  │             │
         │  └─────┬──────┘             │
         │        │                    │
         └────────┴────────────────────┘
```

**Состояния:**
- `WALKING` — движение в направлении
- `FALLING` — падение (проверка высоты для смерти)
- `FLOATING` — спуск с парашютом
- `CLIMBING` — подъём по стене
- `SKILL_ACTIVE` — выполнение навыка (build/bash/mine/dig)
- `BLOCKING` — блокировка прохода
- `EXPLODING` — 5-секундный отсчёт бомбера
- `DYING` — анимация смерти
- `EXITED` — достиг выхода
- `SPLAT` — разбился при падении

### 3.6 Коллизии и физика

- **Ландшафт:** TileMap с коллизиями (StaticBody2D на тайлах)
- **Лемминг:** CharacterBody2D, движение через move_and_slide()
- **Модификация ландшафта:** удаление/замена тайлов через set_cell()
- **Сталь:** отдельный слой тайлов, неразрушаемый
- **Пиксельная точность:** тайлы 16×16 — коллизии на основе тайловой сетки

### 3.7 Автотестирование

**Стратегия тестирования:**

```yaml
Уровни тестирования:
  1. Модульные тесты (GUT - Godot Unit Test):
     - Парсер уровней
     - Конечный автомат лемминга
     - Система навыков
     - Подсчёт очков
  
  2. Интеграционные тесты:
     - Симуляция уровня: запись решений, 
       воспроизведение, проверка результата
     - Проверка что каждый уровень решаем
  
  3. Автоматическое тестирование уровней:
     - Эмуляция touch-событий
     - Скриптованные последовательности навыков
     - Проверка: спасено >= квоты
  
  4. Регрессионное тестирование:
     - CI/CD через GitHub Actions
     - Запуск GUT тестов при каждом PR
```

**Фреймворк:** GUT (Godot Unit Test) — https://github.com/bitwes/Gut

## 5. CI/CD (GitHub Actions)

```yaml
name: Lemmings CI
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    container:
      image: barichello/godot-ci:4.3
    steps:
      - uses: actions/checkout@v4
      - name: Run GUT tests
        run: godot --headless -s addons/gut/gut_cmdln.gd -gdir=res://tests -gexit
```

## 6. Риски и митигация

| Риск | Вероятность | Митигация |
|------|------------|-----------|
| Godot 4.x нестабилен на мобильных | Низкая | Тестирование на реальных устройствах с первой фазы |
| Сложность touch-управления для точных навыков | Средняя | Zoom + замедление времени при назначении навыка |
| Производительность при 100 леммингах | Средняя | Object pooling, оптимизация коллизий |
| Авторские права на дизайн уровней | Низкая | Только свои уровни для прототипа; оригинальные — для изучения |
